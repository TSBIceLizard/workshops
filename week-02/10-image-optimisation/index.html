Image Optimisation Workshop

If a website developer uses a 2000px + 2000px image in their website, but simply reduces it's drawn width on the page either with the HTML or CSS, that said image will continue to take the same amount of time and resources to download from the website onto the end user's cache. It is very much worth considering with raster graphic images be roughly the same width and height as how they are displayed on the page.

Then in the other direction, if a developer chooses to use a very low quality (or low resolution) raster graphic image and stretches it to a huge size, the pixelation will be entirely and egregiously noticable.

Considering this, it's very important to keep the image near 100% its size. Is it a huge image that has its width and breadth scaled down on the page? Consider changing the resolution of the image file to a new file. This can be done with programs like GIMP or Krita (free tools). This will not really work for the other way however, if the developer's image is too small in resolution to nicely fill a space, they should consider re-creating the image or sourcing a new one at relatively the same resolution as the space it's filling!

Image format pros and cons:

  • PNG: Portable Network Graphic. One of the most commonly used and advised image formats of the web. PNG typically sports a compact filesize in most cases and has lossless compression. It supports alpha transparency (holes in the image showing through to what's behind it). Suffix is .png
  • JPEG: Joint Photographic Expert Group. The most popular format on the web but features lossy compression, therfore this format creates alot of artifacting. Suffix is .jpg
  • GIF: Graphics Interchange Format. Gifs support frames of animation if configured to play an animation, but their drawback is their colour depth which is limited to 256 colours per frame. Suffix is .gif
  • WebP: A much newer raster Graphic format created by Google. WebP supports both lossy and lossless compression simultaneously aswel as alpha channel (transparency), including animation. Suffix is .webp
